# Skill 适配评测官

一个帮助用户从大量 Skill 中，筛选出真正适合自己的 Skill 的评测工具。

它不是简单给 Skill 打分，也不是礼貌性夸奖，而是根据用户的身份、目标、能力基础和使用场景，判断一个 Skill 到底是：

- 可以直接使用
- 值得改造
- 值得学习
- 收藏观察
- 不建议投入

一句话说：

**它帮你从一堆 Skill 里，找到真正对你有用的那几个。**

---

## 为什么做它？

现在社区里的 Skill 越来越多。

问题已经不是“有没有 Skill”，而是：

我到底该看哪个？  
哪个对我真的有用？  
哪个值得我花时间研究？  
哪个只是看起来很热闹？

所以我做了这个 Skill。

它想解决的不是“怎么夸一个 Skill”，而是帮用户更快判断：

**这个 Skill 对我有没有用。**

---

## 它适合谁？

适合这些人：

- 想从一堆 Skill 里筛选工具的人
- 想搭建自己 Skill 库的人
- 想判断某个 Skill 值不值得用的人
- 想学习别人 Skill 结构的人
- 想把别人的 Skill 改造成自己工作流的人
- 不想把时间浪费在“看起来很厉害，但其实用不上”的 Skill 上的人

尤其适合：

- 非程序员小白
- AI 编程用户
- 内容创作者
- 产品经理 / 创业者
- HR / 求职服务从业者
- IP / 小说 / AI 漫剧创作者
- 学生 / 学习者
- 想做 Skill 测评的人

---

## 它能做什么？

它会同时判断三件事：

### 1. Skill 本身质量

看这个 Skill 写得好不好，比如：

- 场景是否清楚
- 目标用户是否明确
- 输入输出是否稳定
- 执行流程是否完整
- 有没有判断规则
- 有没有证据意识
- 有没有异常兜底
- 是否可复用、可改造

### 2. 用户适配度

看这个 Skill 对当前用户有没有用，比如：

- 是否解决用户当前任务
- 是否符合用户能力基础
- 是否使用频率够高
- 是否能节省时间、减少返工
- 是否有学习价值
- 是否有改造价值
- 投入产出比是否值得

### 3. 证据与风险

它还会判断：

- 材料是否完整
- 是否只是文本初评
- 有没有真实试用证据
- 是否涉及隐私、账号、财务、法律等风险
- 当前结论是否足够稳

---

## 它怎么评？

它不是上来就打分，而是先做几步判断：

1. 先看用户是谁
2. 再看用户当前想解决什么问题
3. 再看候选 Skill 的材料是否完整
4. 再判断证据等级和风险等级
5. 然后才进入质量评分和适配评分
6. 最后给出行动建议

最终会把 Skill 分成几类：

- **强烈推荐：立刻使用**
- **值得改造**
- **值得学习**
- **收藏观察**
- **不建议投入**

---

## 它和普通测评有什么不同？

普通测评常常只回答：

> 这个 Skill 好不好？

但这个 Skill 更关心：

> 这个 Skill 对你有没有用？

一个 Skill 可能写得很好，但不适合你当前任务。  
一个 Skill 也可能写得一般，但方向很适合你，值得改造成自己的版本。

所以它不是统一排行榜，而是个性化筛选器。

---

## 使用方式

你可以这样对 AI 说：

```text
启动 Skill 适配评测官。

我的身份是：____
我当前目标是：____
我想评测的 Skill 是：____

请帮我判断：
1. 这个 Skill 本身质量怎么样；
2. 它对我有没有用；
3. 我应该直接使用、改造、学习、收藏，还是放弃；
4. 如果要测试它，最小试用任务是什么。
